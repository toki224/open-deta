# バリアナビ プレゼン資料

## アプリケーション概要

**バリアナビ（Barrier Navi）**は、駅のバリアフリー情報を提供し、障害者や高齢者など移動に配慮が必要な方々が駅を利用する際の判断材料を提供するWebアプリケーションです。身体障害・聴覚障害・視覚障害の3つのカテゴリに対応し、各駅のバリアフリー設備を15項目（身体障害）、4項目（聴覚障害）、9項目（視覚障害）で評価し、スコアとして見える化します。

## 詳細機能

### 主要機能
- **スコアリング機能**: 各駅のバリアフリー設備を評価項目に基づいて数値化（例：8/15点）
- **検索・フィルタ機能**: 駅名検索、都道府県フィルタ、バリアフリー設備フィルタ
- **ユーザー認証機能**: ログイン・新規登録、プロフィール管理（お気に入り駅、優先機能設定）
- **ゲストモード**: ログインなしでも利用可能
- **詳細表示機能**: 各駅のバリアフリー設備の詳細情報と基準値との比較表示

### 評価システム
- フラグ型項目（設備の設置有無）、数値型項目（設備の設置数）、割合型項目（設備の適合割合）の3タイプで評価
- 各項目は基準値を満たしているかどうかで「達成（1点）」または「未達成（0点）」を判定
- 達成項目数をカウントし、「達成項目数/総項目数 点」の形式で表示

## 使用技術・構成

### システム構成
- **アーキテクチャ**: クライアント・サーバー型のWebアプリケーション
- **デプロイ方式**: Docker Composeによるマルチコンテナ構成
- **バックエンド**: Python 3.11 + Flask（RESTful API）
- **フロントエンド**: TypeScript + HTML5 + CSS3
- **データベース**: MySQL 8.0（Dockerコンテナ）

**コンテナ・インフラ**

- Dockerネットワーク（bridgeネットワークによるコンテナ間通信）
- Dockerボリューム（データ永続化）
- ヘルスチェック機能（データベース起動待機）

**バックエンド**
- Python 3.11（ベースイメージ: python:3.11-slim）
- Flask（Webフレームワーク、RESTful API）
- Flask-CORS（CORS対応）
- mysql-connector-python（MySQL接続）
- bcrypt（パスワードハッシュ化）
- python-dotenv（環境変数管理）

**フロントエンド**
- TypeScript（型安全なJavaScript）
- Node.js 18.x（TypeScriptコンパイル用、Dockerコンテナ内にインストール）
- HTML5 / CSS3

**データベース**
- MySQL 8.0（Docker公式イメージ）
- stationsテーブル（駅のバリアフリー設備情報）
- usersテーブル（ユーザー情報）
- users_preferencesテーブル（ユーザー設定情報）

### Docker構成の詳細

**コンテナ構成**
- **webコンテナ**: Flask APIサーバー
  - Python 3.11 + Flaskアプリケーション
  - TypeScriptコンパイル環境（Node.js 18.x）
  - 自動起動スクリプト（docker-entrypoint.sh）
  - MySQL接続待機機能
  - CSVデータ自動インポート機能
  
- **dbコンテナ**: MySQLデータベースサーバー
  - MySQL 8.0

  - ヘルスチェック機能

**ネットワーク構成**
- カスタムブリッジネットワーク（barrier_navi_network）
- コンテナ間通信によるサービス連携
- ポートマッピング（ホスト:コンテナ）
  - Flask: 5000:5000
  - MySQL: 3307:3306（デフォルト、環境変数で変更可能）

**データ管理**
- Dockerボリュームによるデータ永続化（mysql_data）
- 環境変数ファイル（.env）による設定管理
- CSVファイルマウントによるデータインポート
- ログファイルマウント（logsディレクトリ）

**自動化機能**
- データベース初期化の自動実行
- CSVデータの自動インポート
- MySQL接続待機（最大30回リトライ）
- TypeScriptの自動コンパイル（Dockerイメージビルド時）

## 評価

### 1. 課題解決性 ⭐⭐⭐⭐⭐

**高い課題解決性を有する**

バリアナビは、障害者や高齢者の「駅を利用する際の情報不足」という社会的課題に直接的に取り組んでいます。

- **具体的な課題**: 障害者や高齢者が駅を利用する際、事前にバリアフリー設備の有無を確認することが困難で、実際に駅に到着してから利用できないことが判明するケースが多い
- **解決アプローチ**: 
  - 駅のバリアフリー設備情報をデータベース化し、Web上で検索・閲覧可能に
  - 複数の評価項目をスコア化することで、駅同士の比較を容易に
  - 障害カテゴリごとに最適化された評価項目を提供
- **実用性**: 実際の駅データ（東京の駅データなど）を使用し、現実的な情報提供が可能
- **対象ユーザー**: 身体障害者、聴覚障害者、視覚障害者、高齢者など、移動に配慮が必要な幅広い層

### 2. 新規性 ⭐⭐⭐⭐

**一定の新規性を有する**

既存のバリアフリー情報提供サービスと比較して、以下の点で新規性があります。

- **スコア化による見える化**: 単なる設備の有無の羅列ではなく、複数の評価項目を統合したスコア（例：8/15点）として数値化することで、駅のバリアフリー度を直感的に理解可能
- **障害カテゴリ別の最適化**: 身体障害・聴覚障害・視覚障害の3カテゴリに対応し、それぞれに最適化された評価項目を提供（身体障害15項目、聴覚障害4項目、視覚障害9項目）
- **基準値に基づく客観的評価**: 各評価項目に基準値を設定し、客観的な評価基準を確立（例：エレベーター4基以上、プラットホーム6以上など）
- **ユーザーカスタマイズ機能**: お気に入り駅の登録、優先したい機能の設定など、個人のニーズに合わせたカスタマイズが可能

**改善の余地**: 
- 既存のバリアフリー情報提供サービス（国土交通省のバリアフリー情報提供システムなど）との差別化をより明確にする必要がある
- 重み付け設定機能（優先機能に基づくスコア調整）は未実装のため、今後の実装により新規性が向上する可能性がある

### 3. 論理性 ⭐⭐⭐⭐⭐

**高い論理性を有する**

評価システムの設計から実装まで、一貫した論理的なアプローチが取られています。

- **評価項目の体系化**: 
  - フラグ型（設備の設置有無）、数値型（設備の設置数）、割合型（設備の適合割合）の3タイプに分類
  - 各タイプに応じた明確な評価基準を設定
- **基準値の根拠**: 
  - 各数値型項目に基準値（例：エレベーター4基以上、プラットホーム6以上）を設定
  - 基準値は身体障害者が実際に駅を利用する上で必要な設備の充実度を考慮
- **スコア計算の透明性**: 
  - 各項目が「達成（1点）」または「未達成（0点）」の2値で評価される明確なルール
  - 達成項目数をカウントし、「達成項目数/総項目数 点」の形式で表示することで、計算過程が透明
- **実装の一貫性**: 
  - バックエンドでスコア計算ロジックを実装し、フロントエンドは表示のみを担当する役割分担が明確
  - API設計がRESTfulで、エンドポイントの命名規則が統一されている
- **データ構造の合理性**: 
  - MySQLデータベースに駅データを構造化して保存
  - ユーザー情報と設定情報を分離したテーブル設計

**改善の余地**: 
- 重み付け設定機能が未実装のため、ユーザーの優先機能に基づくスコア調整ができない（論理的には実装可能だが、現時点では未対応）

## 総合評価

バリアナビは、**課題解決性、新規性、論理性のすべてにおいて高い評価を得られる**アプリケーションです。特に、社会的課題への取り組み姿勢と、評価システムの論理的な設計が強みです。今後、重み付け設定機能の実装や、既存サービスとの差別化をより明確にすることで、さらなる価値向上が期待できます。
